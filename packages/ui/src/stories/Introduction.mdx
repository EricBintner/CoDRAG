# CoDRAG UI Design System

Welcome to the CoDRAG UI component library and design system.

## Overview

This Storybook documents the shared UI components used across:
- **CoDRAG Dashboard** — The main application UI
- **CoDRAG Website** — Marketing site and documentation

## Design Foundations

Before diving into components, review the design foundations:

### Visual Direction
The visual direction determines the overall look and feel. Review the options in:
`docs/Phase13_Storybook/theme-examples/`

Available directions:
- **Direction A: Slate Developer** — Clean, professional, neutral
- **Direction B: Deep Focus** — Dark, immersive, code-centric
- **Direction C: Signal Green** — Technical, status-forward
- **Direction D: Warm Craft** — Approachable, human, indie-dev feel

### Design Tokens
All visual properties are defined as CSS custom properties (design tokens):
- Colors (background, surface, text, semantic)
- Typography (font families, sizes, weights)
- Spacing (consistent 4px-based scale)
- Border radius
- Shadows
- Motion/animation

See `src/tokens/index.css` for the full token set.

## Component Categories

### Status Components
Components that communicate system state:
- `StatusBadge` — Fresh/Stale/Building indicators
- `StatusCard` — Project status overview

### Search Components
Components for search functionality:
- `SearchResult` — Individual search result row
- `CodeChunk` — Code snippet with syntax highlighting
- `CitationBlock` — Source citation display

### Navigation Components
Components for app navigation:
- `Sidebar` — Project list and navigation
- `Tabs` — Project/view switching

### Form Components
Components for user input:
- `Input` — Text input
- `Toggle` — Boolean toggle
- `Select` — Dropdown select

## Tech Stack

- **React 18** — Component framework
- **TypeScript** — Type safety
- **Tailwind CSS** — Utility-first styling
- **Tremor** — Dashboard component library
- **Radix UI** — Accessible primitives
- **Lucide** — Icon library

## Getting Started

```bash
# Install dependencies
cd packages/ui
npm install

# Run Storybook
npm run storybook
```

## Contributing

When adding new components:
1. Create the component in `src/components/`
2. Add stories in `src/stories/` or co-located `.stories.tsx`
3. Document props with JSDoc comments
4. Include loading, empty, and error states
5. Test accessibility with keyboard navigation
